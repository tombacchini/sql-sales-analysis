# Analyse des ventes avec SQL

## Présentation

À travers ce projet, j'analyse des données de ventes à l'aide de SQL dans le but d'évaluer la performance commerciale de l'entreprise d'un Dataset fictif créé dans le cadre d’un projet d’apprentissage. 

## Objectifs métier

- Mesurer le chiffre d’affaires global
- Analyser l’évolution du chiffre d’affaires dans le temps
- Identifier les clients les plus performants
- Détecter les produits les moins rentables
- Comprendre la répartition du chiffre d’affaires par région
- Évaluer la performance des différentes catégories de produits

##  Outils utilisés

- PostgreSQL
- DBeaver
- SQL

## Description des données

Le dataset utilisé nommé "sales_financial" contient les informations suivantes:

- 'client_name' : nom du client
- 'product_name' : produit vendu
- 'category' : catégorie du produit
- 'region' : zone géographique
- 'sale_date' : date de vente
- 'net_revenue' : chiffre d’affaires
- 'gross_profit' : profit généré
- 'margin_pct' : marge en pourcentage

## Analyses réalisées

### 1. Chiffre d’affaires total
Permet d’évaluer la performance globale de l’entreprise.

### 2. Chiffre d’affaires mensuel
Analyse l’évolution du chiffre d’affaires dans le temps.

# 3. Évolution mensuelle
Mesure la variation du chiffre d’affaires d’un mois à l’autre grâce aux fonctions de fenêtre.

# 4. Produits les moins rentables
Identifie les produits avec les marges les plus faibles.

# 5. Top 5 clients
Met en évidence les clients générant le plus de chiffre d’affaires.

# 6. Clients au-dessus de la moyenne
Identifie les clients dont la performance dépasse la moyenne globale.

# 7. Répartition du CA par région
Analyse la contribution de chaque région au chiffre d’affaires total.

# 8. Performance par catégorie
Compare le chiffre d’affaires, le profit et la marge selon les catégories de produits.

# Conclusion
Ce projet m’a permis de mettre en pratique SQL sur un cas concret et de structurer une analyse de données orientée métier.
